6 张图：5 张层内流转（L1-L5）+ 1 张跨层流转。
所有逆向操作 = append 模式：上游记录永不删除（标 deprecated/superseded），下游标 drifted，各自 history[] 留痕完整可追溯。
图例： 实线绿粗箭头 = 正向 action    虚线红箭头 = 逆向 action（append 模式，标 deprecated/superseded，不删）    虚线橙箭头 = 层间跳转下沉    紫色 = 影响传播 / L5 effort    黄色 = history[] append-only 数据 / 说明
逆向三原则： ① 上游 append 不 delete，旧记录标 deprecated/superseded 保留可追溯    ② 下游 drift 是持久状态，写入时触发标记（drifted 阻塞 closeout gate）    ③ 所有状态变化进对应 history[] 字段（lockHistory / splitHistory / ticketHistory / driftLog 等）
